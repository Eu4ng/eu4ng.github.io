## 개요

[adamrehn/ue4-docker](https://github.com/adamrehn/ue4-docker)를 활용한 **UE 5.7 Windows 개발 이미지** 빌드 과정을 정리한 문서입니다.

## 서론

**언리얼 엔진 CI/CD 파이프라인**을 구축하려면 **Windows 개발 이미지**가 필수적입니다. 

**Epic Games**에서 공식적으로 제공하는 [리눅스용 이미지](https://github.com/orgs/EpicGames/packages/container/package/unreal-engine)는 존재하지만, **Windows 패키징**이 필요한 환경에서는 [공식 가이드](https://dev.epicgames.com/documentation/unreal-engine/building-the-windows-container-images-from-source)를 따르더라도 **런타임 이미지**만 빌드 가능합니다. 따라서 **Windows 개발 이미지**를 사용하려면 직접 빌드 과정을 거쳐야 합니다.

## 환경 설정

> 이미 환경이 구축되어 있는 경우 건너뛰셔도 됩니다.
{: .prompt-info}

### 1. Python 설치

설정 문제를 방지하기 위해 **Python install manager** 대신 **Standalone Installer** 사용을 권장합니다.

- [Python 공식 홈페이지](https://www.python.org/downloads/)에서 다운로드 및 설치
  - **설치 옵션**:
    - `Use admin privileges when installing py.exe`: **선택**
    - `Add python.exe to PATH`: **선택**
  - **설치 완료 후**:
    - `Disable path length limit`: **클릭**

### 2. Git 설치

**Standalone Installer**로 설치해야 합니다.

- [Git 공식 홈페이지](https://git-scm.com/install/windows)에서 다운로드 및 설치

### 3. Hyper-V 및 컨테이너 기능 활성화

> **Hyper-V** 기능은 **Windows Pro** 이상의 에디션에서만 제공됩니다. **Hyper-V** 기능이 활성화되어 있어야 시스템 트레이에서 **Docker Desktop** 아이콘 우클릭 시 `Switch to Windows containers` 메뉴가 활성화 됩니다.
{: .prompt-warning}

- **Windows 기능 켜기/끄기**에서 다음 항목 활성화:
  - `Hyper-V`
  - `가상 머신 플랫폼`
  - `컨테이너`

### 4. WSL 2 설치 (선택 사항)

**WSL 2**는 주로 리눅스 컨테이너에서 사용되지만, **Docker Desktop**의 기본 구동 기반이 되므로 함께 설치하는 것이 좋습니다.

- **Windows 기능 켜기/끄기**에서 다음 항목 활성화:
  - `Linux용 Windows 하위 시스템`

- **명령 프롬프트**에서 실행:
  ```shell
  wsl --install
  ```

### 5. Docker Desktop 설치

> **용량 주의**: 빌드된 최종 이미지는 약 80GB 내외지만, 빌드 과정에서의 임시 파일과 중간 레이어를 포함하면 **800GB 이상의 여유 공간**이 필요할 수 있습니다.
{: .prompt-danger}

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) 다운로드 및 관리자 권한으로 설치
  - `Use WSL 2 instead of Hyper-V (recommended)`: **체크 해제**
  - `Allow Windows Containers to be used with this installation`: **체크**
- **Docker Desktop** 실행 후 **Privileged helper service is not running** 오류 발생 시 우측 하단의 **Start Service** 클릭
- 설치 후 시스템 트레이에서 **Docker Desktop** 아이콘 우클릭 후 **Switch to Windows containers** 클릭
- **Settings > Docker Engine**에서 저장소 경로(`data-root`)와 최대 용량을 설정합니다:

  ```json
  {
    "data-root": "D:\\Docker",
    "experimental": false,
    "storage-opts": [
      "size=800GB"
    ]
  }
  ```

### 6. ue4-docker 패키지 설치

> **주의**: 현재 공식 **ue4-docker** 패키지는 [UE 5.6 이상 버전 빌드 이슈](https://github.com/adamrehn/ue4-docker/issues/381)가 있습니다. 따라서 본 게시글에서는 임시 조치된 아래 패키지를 설치하여 진행하였습니다.
{: .prompt-warning}

```shell
# 공식 패키지 설치
pip install ue4-docker

# 임시 조치된 패키지 설치
pip install git+https://github.com/E4-Unreal/ue4-docker.git@develop
```

## UE 5.7.4 이미지 빌드

### 1. 빌드 실행

> **성능 주의**: `--isolation=process` 옵션을 사용해야 모든 CPU 코어를 활용해 빌드 속도를 극대화할 수 있습니다. 만약 파일 접근 권한 오류(`0x20`)가 발생한다면 어쩔 수 없이 `--isolation=hyperv`를 사용해야 하며, 이 경우 빌드 시간이 대폭 늘어납니다.
{: .prompt-warning}

> 처음 빌드 시 **Python** 관련 권한 요청으로 인해 **GitHub** 로그인 관련 오류가 발생하며 중단될 수 있습니다. 수락 후 다시 시도하면 됩니다.
{ .prompt-warning }

**명령 프롬프트**에서 다음 명령어를 입력합니다.

```shell
ue4-docker build 5.7.4 --target minimal --exclude debug --exclude templates --visual-studio 2022 -basetag=ltsc2022 -isolation=process -username [GITHUB_USERNAME] -password [PAT]
```

빌드가 성공적으로 진행된다면 중간에 `adamrehn/ue4-build-prerequisites`, `adamrehn/ue4-source` 이미지가 순차적으로 생성되고 마지막으로 `adamrehn/ue4-minimal` 이미지가 생성됩니다.

빌드 시간은 사양에 따라 다르며, 제 환경에서는 약 **12시간**이 소요되었습니다. 다음은 빌드 시 컴퓨터 사양 및 로그 마지막 부분입니다.

```shell
# 컴퓨터 사양
# - AMD Ryzen 7 7800X3D 8-Core / 16-Thread
# - RAM DDR5 5600MHz 64GB
# - Windows 11 Pro 버전 25H2 (OS 빌드 26200.8246)
# - Docker Desktop 4.70.0 (224270)
# - UE 5.7.4

[ue4-docker build] Built image "adamrehn/ue4-minimal:5.7.4-ltsc2022-vs2022" in 11 hours, 19 minutes and 33.44 seconds

[ue4-docker build] Skipping ue4-full image build.

[ue4-docker build] Total execution time: 12 hours, 27 minutes and 31.87 seconds
```

### 2. 이미지 배포

> > **법적 주의**: **언리얼 엔진 Windows 개발 이미지**는 기술적/법적 제한으로 인해 **공개 배포가 금지**되어 있습니다. 반드시 **조직 내부 전용(Private) 레지스트리**에만 배포하시길 바랍니다. 자세한 사항은 [공식 문서](https://dev.epicgames.com/documentation/unreal-engine/known-limitations-of-containers-in-unreal-engine)를 살펴보시길 바랍니다.
{: .prompt-danger}

저는 **GitLab Self-Managed** 서버를 사용하고 있으며 다음과 같이 배포했습니다.

```shell
# GitLab Container Registry 로그인
docker login [GITLAB_DOMAIN]

# 이미지 태그 변경
docker tag adamrehn/ue4-minimal:5.7.4 [GITLAB_DOMAIN]/[USERNAME]/unreal-engine:dev-minimal-windows-5.7.4
docker tag adamrehn/ue4-minimal:5.7.4-ltsc2022-vs2022 [GITLAB_DOMAIN]/[USERNAME]/unreal-engine:dev-minimal-windows-5.7.4-ltsc2022-vs2022

# 이미지 푸시
docker push [GITLAB_DOMAIN]/[GROUP]/[PROJECT]/unreal-engine:dev-minimal-windows-5.7.4
docker push [GITLAB_DOMAIN]/[GROUP]/[PROJECT]/unreal-engine:dev-minimal-windows-5.7.4-ltsc2022-vs2022
```

## 트러블 슈팅

### 1. For security reasons Docker Desktop must be owned by an elevated account

**Docker Desktop** 설치 시 관리자 권한으로 실행하시면 됩니다.

### 2. "Platform Win64 is not valid" (UE 5.6+)

마이크로소프트에서 **Windows10SDK**를 **Windows11SDK**로 이름을 변경하였기 때문에 발생하는 이슈로, `ue4docker/dockerfiles/ue4-build-prerequisites/windows/install-prerequisites.ps1` 파일 수정을 통해 임시 조치하였습니다.

자세한 사항은 [Platform Win64 is not valid for UE 5.6 이슈](https://github.com/adamrehn/ue4-docker/issues/381) 이슈에서 확인 가능합니다.

### 3. hcsshim::ActivateLayer failed in Win32: The process cannot access the file because it is being used by another process. (0x20)

데스크탑에서는 발생하지 않았지만 미니 PC(Ryzen 8845HS)에서 빌드 시 발생한 오류입니다.

결론부터 말씀드리면 `-isolation=hyperv` 옵션을 사용하면 빌드가 정상적으로 되지만, `-isolation=process` 옵션에서는 해결 방법을 찾지 못하였습니다.

[Windows-specific issues \| ue4-docker user manual](https://ue4-docker.docs.adamrehn.com/#building-the-engine-in-a-windows-container-fails-with-the-message-the-process-cannot-access-the-file-because-it-is-being-used-by-another-process) 문서에 따르면 메모리가 **4GB**의 배수가 아니라면 발생하는 이슈라고 합니다. 제 미니 PC의 메모리는 **32GB**이지만 그래픽 전용 메모리 할당으로 **3GB** 사용중이기에 **29GB**로 인식되어 **4GB**의 배수가 아니라서 발생하는 것으로 추정됩니다.

BIOS에서 그래픽 메모리 할당을 **2GB**나 **4GB**로 설정하면 문제가 해결될 수도 있을 것 같습니다만, 시간 관계상 추후에 확인해보도록 하겠습니다.
