## 개요

맥 미니에 **OpenClaw**를 설치하고 **로컬 LLM**과 **Discord**를 연동하는 과정을 정리한 문서입니다.

## 사전 준비

> 이미 설치되어 있는 경우 건너뛰셔도 됩니다.
{: .prompt-info }

> **Homebrew(brew)를 통한 설치를 강력히 권장드립니다.**  
> 공식 홈페이지의 `.pkg` 파일을 이용해 설치를 진행할 경우, OpenClaw 설치 과정에서 권한 문제가 발생하여 진행이 막힐 수 있습니다.
{: .prompt-warning }

### 1. Homebrew 설치

```bash
# Homebrew 설치
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# 경로 설정
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/사용자이름/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"

# 설치 확인
brew --version
```

### 2. Node.js 설치

```bash
# Node.js 설치
brew install node
```

### 3. Ollama 설치

```bash
# Ollama 설치
brew install ollama

# 모델 다운로드
ollama pull gemma4:e2b

# 서비스 등록 및 실행 확인
brew services start ollama
curl http://localhost:11434
```

### 4. Discord 봇 생성
1. [Discord 개발자 포털](https://discord.com/developers/applications)에서 신규 애플리케이션을 생성합니다.
2. `봇 > 토큰` 항목의 `토큰 초기화` 버튼을 클릭하여 토큰을 생성하고 따로 메모해 둡니다.
3. `봇 > Privileged Gateway Intents` 항목을 모두 활성화합니다.
    - Presence Intent
    - Server Members Intent
    - Message Content Intent
4. `OAuth2 > OAuth2 URL 재생기` 항목에서 아래 옵션들을 체크하여 초대 URL을 생성합니다.
    - **Scopes**: `bot`, `applications.commands`
    - **Bot Permissions**: `View Channels`, `Send Messages`, `Embed Links`, `Attach Files`, `Read Message History`, `Add Reactions`
5. 생성된 URL로 봇을 본인의 서버에 초대하고, 전용 채팅 채널(`openclaw`)을 생성합니다.

## OpenClaw 설치 및 설정

### 1. OpenClaw 설치
```bash
# OpenClaw 전역 설치
npm i -g openclaw

# 초기 설정 시작
openclaw onboard
```

### 2. OpenClaw 설정

> 화살표 키로 이동, 스페이스로 선택, 엔터로 확정합니다.
{: .prompt-tip }

`openclaw onboard` 실행 후 나타나는 질문들에 다음과 같이 응답하세요.

| 질문 항목 | 선택 및 입력 값 |
| :--- | :--- |
| **Setup mode** | `QuickStart` |
| **Model/auth provider** | `Ollama` |
| **Ollama mode** | `Local only` |
| **Ollama base URL** | `http://127.0.0.1:11434` |
| **Default model** | `ollama/gemma4:e2b` |
| **Select channel** | `Discord (Bot API)` |
| **Enter Discord bot token** | 메모해둔 토큰 입력 |
| **Discord channels access** | `Allowlist (recommended)` |
| **Discord channels allowlist** | `openclaw` |
| **Search provider** | `Ollama Web Search` |
| **Hatch your bot** | `Hatch in Terminal` |

## 채널 연동 확인 및 트러블슈팅

### 1. 디스코드 봇 페어링 설정

처음 디스코드 봇에게 DM을 보내면 접근 권한이 없다는 답장이 옵니다.

메시지 하단에 표시된 `openclaw pairing approve discord [PARSING_CODE]` 형태의 명령어를 복사하여 터미널에 입력하면 페어링이 완료됩니다.

### 2. 멘션 없이 대화하기

기본적으로 디스코드 채팅 채널에서는 멘션과 함께 메시지를 입력해야 답장을 합니다. 멘션 없이 지정된 채널에서 바로 대화하고 싶다면 아래 과정을 거쳐야 합니다.

**[설정 방법]**

1. `오픈클로 대시보드 > 채널 > Guilds > Require Mention`을 **활성화**하고 저장합니다.
2. 다시 해당 항목을 **비활성화**하고 저장합니다.

> **왜 이렇게 하나요?**  
> 대시보드 기본값은 비활성화 상태이지만, 실제 설정 파일(`openclaw.json`)에 값이 명시되지 않아 활성화된 것처럼 동작하는 버그가 있습니다. 위와 같이 수동으로 한 번 켰다 꺼주면 설정이 정상적으로 저장됩니다.
{: .prompt-info }

직접 파일을 수정하려면 `vi ~/.openclaw/openclaw.json`를 통해 아래 형식처럼 작성하시면 됩니다.
```json
  "channels": {
    "discord": {
      "enabled": true,
      "token": "TOKEN",
      "groupPolicy": "allowlist",
      "guilds": {
        "*": {
          "requireMention": false
        }
      }
    }
  }
```

## 마무리

이상으로 맥 미니에 **OpenClaw**를 설치하고 **로컬 LLM**과 **Discord**를 연동하는 과정을 알아보았습니다.

여담으로 저는 **M4 맥 미니 16GB 모델**에 설치를 진행하였는데 현재로서는 **로컬 LLM**을 운영하기에는 적합하지 않은 것 같습니다. 고사양 모델인 `gemma4:26b`는 **메모리**가 부족하여 심각할 정도로 매우 느리고, 저사양 모델인 `gemma4:e2b`조차 **GPU 처리 속도** 때문에 답답하다고 느껴졌습니다.

만약 비슷한 사양의 서버를 운영중이시라면 **로컬 LLM**은 테스트 느낌으로만 사용하시는 것을 추천드립니다.
